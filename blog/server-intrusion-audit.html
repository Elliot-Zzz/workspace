服务器被入侵后如何排查：日志分析完整指南

2026-03-22 · 约 20 分钟阅读


Elaine

💡 背景
今天早上 6 点，我们的阿里云服务器被暴力破解了 root 密码。本文记录完整的排查思路和命令，供大家学习和参考。

📋 排查清单速览

发现服务器被入侵后，按这个顺序检查最有效率：

  1. 断开网络 / 停止对外服务（如果情况严重）
  2. 查看登录记录——谁在什么时候登录的？
  3. 查看命令历史——他们执行了什么？
  4. 查看网络连接——有没有正在进行的远程控制？
  5. 查看用户列表——有没有新增的管理员账号？
  6. 查看定时任务——有没有定时任务在搞事？
  7. 查看服务列表——有没有新增的可疑服务？
  8. 查看安装历史——有没有装新软件？
  9. 查看启动项——有没有开机自启的后门？
  10. 查看文件改动——关键配置文件有没有被改？

1️⃣ 查看登录记录

登录记录是最重要的突破口。可以告诉你：谁、从哪个IP、什么时间登录的、待了多久。

查看成功登录记录

last

输出示例：

elliot   pts/0    139.226.130.70   Tue Mar 17 18:15 - 20:27  (02:11)
root     pts/0    39.97.253.143    Sun Mar 22 06:08 - 06:08  (00:00)
reboot   system boot  6.8.0-100-generi Wed Mar 18 12:12   still running

重点关注：陌生的 IP 地址、非正常时段的登录、root 账号的异常登录。

过滤特定IP段（排除自己的正常IP）

# 查看非正常IP的成功登录（把你的IP段替换进去）
last | grep -v "139.226\|139.227\|223.166\|你的IP段"

查看失败登录记录

sudo lastb

这个命令列出所有失败的登录尝试。攻击者在密码被改之前会一直尝试，看到大量失败记录很正常。

查看 auth.log 中的 Accepted 记录

sudo grep "Accepted" /var/log/auth.log
sudo grep "Accepted" /var/log/auth.log.1    # 昨天的日志

这是最直接的——只要看到 Accepted password for root from 陌生IP，就说明有人成功登录了。

限定时间范围查看日志

# 查看某个时间段的所有认证记录
sudo journalctl --since "2026-03-22 06:00" --until "2026-03-22 09:30" | grep -i "accepted\|failed"

journalctl 是 systemd 日志的查看工具，比直接读文件更方便，支持时间范围过滤。

2️⃣ 查看命令历史

攻击者登录后通常会执行一些命令来探测环境。命令历史是重要的突破口。

查看 bash 历史

sudo cat /root/.bash_history
⚠️ 注意
如果攻击者执行了 rm -rf ~/.bash_history 或设置了 HISTORY=0，历史记录会被清空。清空本身就是一个危险信号！

关键词过滤

# 搜索可疑命令关键词
grep -E "wget|curl|chmod|useradd|deluser|ssh|scp|tar|zip|rm |python|perl|bash" /root/.bash_history

攻击者常做的事：下载工具（wget/curl）、改权限（chmod）、创建账号（useradd）、内网渗透（ssh）、下载后门（python/perl 脚本）。

3️⃣ 查看网络连接

如果攻击者正在连接你的服务器，或者服务器正在向外发起连接，这些都能看到。

查看所有 TCP/UDP 连接

sudo ss -tunap

参数说明：-t TCP，-u UDP，-n 显示IP而非域名，-a 所有，-p 显示进程

重点关注

  • ESTABLISHED — 正在进行的连接，尤其是对外部IP的
  • LISTEN — 服务器在监听的端口，有没有新增的监听端口
  • 陌生的外部 IP，尤其是连接到一些非常见端口（如 4444、5555、6666 等 Metasploit 默认端口）

查看端口对应的进程

sudo lsof -i :端口号

比如 sudo lsof -i :5901 可以查看 VNC 端口被哪个进程占用。

检查对外连接

# 查看服务器主动向外建立的连接（排除正常内网通信）
sudo ss -tunap | grep ESTAB | grep -v "127.0.0.1\|172.21\|100.100\|内网IP段"

4️⃣ 查看用户列表

攻击者入侵后通常会创建一个自己的管理员账号，方便下次回来。

查看所有用户

cat /etc/passwd | grep -v "^_" | grep "sh$"

只显示有 shell 的用户（能登录的账号）。重点关注陌生的账号。

查看新增用户

# 查看 UID >= 1000 的用户（正常新建用户从这个UID开始）
 awk -F: '{ if($3 >= 1000) print $1, $3 }' /etc/passwd

查看用户组（有没有被加进 sudo 或 root 组）

getent group sudo
getent group root
getent group admin

查看 shadow 文件（看哪些账号有密码）

sudo cat /etc/shadow | grep -v "!" | grep -v ":\*$:"

如果看到陌生账号有密码 hash，说明这个账号可以被登录。

5️⃣ 查看定时任务

定时任务（cron job）是保持持久化最常用的手段。攻击者会设置一个定时任务，每分钟或每小时执行一次后门程序。

查看当前用户的 cron

crontab -l
sudo crontab -l

查看系统级 cron 任务

sudo cat /var/spool/cron/crontabs/root
sudo ls -la /etc/cron.d/
sudo ls -la /etc/cron.daily/ /etc/cron.hourly/ /etc/cron.monthly/

查看 systemd 定时器（timer）

systemctl list-timers --all

除了 cron，systemd 也有自己的定时器机制。

关键词搜索

# 在 cron 配置里搜索可疑关键词
grep -r "wget\|curl\|python\|bash\|sh " /etc/cron* /var/spool/cron/ 2>/dev/null

6️⃣ 查看服务列表

攻击者可能会在服务器上安装一个新的服务，开机自动运行，比 cron 更隐蔽。

查看正在运行的服务

systemctl list-units --type=service --state=running

重点关注：名字奇怪的、不认识的、非系统自带的。

查看所有服务（含未启动的）

systemctl list-units --type=service --all | grep -v "loaded\|systemd\|ssh\|cron\|network"

检查服务状态

systemctl status 服务名
sudo systemctl status 可疑服务名

7️⃣ 查看软件安装历史

如果攻击者用包管理器安装了工具，可以通过安装历史查到。

apt 安装历史（Debian/Ubuntu）

cat /var/log/apt/history.log | tail -100
cat /var/log/apt/history.log.1.gz | zcat | tail -100

可以看到什么时间安装了什么包。如果发现可疑的软件（如 nmap、netcat、hydra 等黑客工具），要警惕。

查看 dpkg 安装记录

dpkg -l | grep "^ii" | tail -50

列出所有已安装的包。重点关注不认识的包。

yum/dnf 安装历史（CentOS/RHEL）

cat /var/log/yum.log
sudo yum history

8️⃣ 查看文件改动

如果攻击者修改了系统文件（如 /etc/passwd、/etc/shadow、/etc/sudoers），可以通过文件的修改时间查到。

查看某个时间点附近修改的文件

# 查找 2026-03-22 06:00 到 09:30 之间修改的系统文件
sudo find /etc /usr/bin /usr/sbin /bin /sbin -type f -newer /var/log/auth.log.1 -mmin -1200 2>/dev/null | grep -v "sysstat\|e2scrub"

查看敏感文件的修改时间

ls -la /etc/passwd /etc/shadow /etc/sudoers /etc/ssh/sshd_config
stat /etc/passwd

使用 AIDE 进行完整性检测（推荐安装）

# AIDE 是一个文件完整性监控工具
sudo apt install aide
sudo aideinit

平时运行一次 aide --check 就能发现哪些文件被改过了。建议在服务器刚部署好时就建立基准。

使用 rkhunter 检测 Rootkit

sudo apt install rkhunter
sudo rkhunter --check

rkhunter 会检查常见的 Rootkit 特征、隐藏进程、可疑网络连接等。

9️⃣ 查看 Docker 容器

攻击者可能利用 Docker 来运行后门或挖矿程序。

# 查看正在运行的容器
docker ps -a

# 查看所有镜像
docker images

# 查看容器日志（如果发现可疑容器）
docker logs 容器ID

本次排查中，Docker 没有运行任何容器，也没有可疑镜像。

🔟 systemd journal 高级查询

journalctl 是查看 Linux 系统日志的利器，功能比直接读文件强大得多。

按时间范围查询

# 查看今天凌晨到现在的所有日志
journalctl --since "today"

# 查看某个时间段的日志
journalctl --since "2026-03-22 06:00" --until "2026-03-22 09:30"

# 查看昨天的日志
journalctl --since "2026-03-21 00:00" --until "2026-03-22 00:00"

按服务过滤

# 只看 sshd 的日志
journalctl -u sshd --since "2026-03-22 06:00"

# 只看 cron 的日志
journalctl -u cron --since "today"

关键词搜索

# 搜索包含 "error" 的日志
journalctl --since "today" | grep -i error

# 搜索多个关键词
journalctl --since "2026-03-22 06:00" | grep -E "wget|curl|useradd|chmod|sudo"

实时跟踪日志

# 等效于 tail -f，实时监控新日志
journalctl -f

🚨 发现入侵后的应急响应

  1. 断网 — 如果情况严重，立即断开服务器网络连接，防止攻击者继续操作或对外攻击
  2. 截图/录屏 — 先把当前屏幕截图、日志输出保存，作为证据
  3. 停止可疑服务 — systemctl stop 可疑服务名
  4. 删除攻击者账号 — userdel -r 用户名
  5. 删除攻击者的定时任务 — crontab -r -u 用户名
  6. 删除攻击者的 SSH 密钥 — 检查 ~/.ssh/authorized_keys，移除不认识公钥
  7. 更改所有密码 — root、数据库、应用密码等全部改掉
  8. 查日志确认攻击范围 — 确定攻击者到底做了什么
  9. 快照备份 — 在清理前创建快照，以防万一
  10. 重装系统 — 如果不确定清理是否干净，最稳妥的办法是重装

📚 推荐工具

  • AIDE — 文件完整性监控，建立基准后能发现任何文件改动
  • rkhunter — Rootkit 检测，扫描常见的后门特征
  • fail2ban — 自动封禁多次登录失败的 IP
  • lynis — Linux 安全审计工具，扫描系统漏洞和配置问题
  • ClamAV — 恶意软件扫描器，可以扫描服务器上的病毒和后门

📝 总结

排查服务器入侵是一个系统工程，重点是：

  • 先看登录记录 — 确认攻击者是谁、从哪来、什么时候来的
  • 再看命令历史 — 确认攻击者做了什么
  • 然后排查后门 — 用户、定时任务、服务、文件改动
  • 最后清理加固 — 删账号、改密码、关漏洞

最重要的是：平时就要做好安全措施，而不是等到被入侵才想起来。希望这篇文章对你有帮助，祝大家的服务器都平安！🛡️
