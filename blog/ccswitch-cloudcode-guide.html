🔍 概念澄清：Cloud Code 是什么？

「Cloud Code」在不同语境下有两个含义，需要先厘清：

  • Google Cloud Code（VS Code 官方扩展）：Google 官方出品的 VS Code 扩展，专注于 Google Cloud 开发，支持 Cloud Run、Cloud Build、Kubernetes 等场景，不直接提供 AI 编程能力。
  • Cloud Code = Claude Code（在中文社区的常见说法）：在中文开发者社区中，「Cloud Code」常被用来指代 Claude Code——Anthropic 出品的 AI 编程 Agent 工具，可以集成到 VS Code（通过 VS Code Insiders 的 Chat 接口）或独立终端使用。这也是本文的重点。

本文以 Claude Code 为主线，搭配 CCSwitch 实现国产大模型的灵活切换，覆盖 Skills、Commands、Hooks、Agents 全套工作流。

🔌 CCSwitch 完全解析

CCSwitch（farion1231/cc-switch）是一个跨平台桌面工具，用 Rust + React/Tauri 构建，用于统一管理多个 AI 编码 CLI 的配置。它支持的应用包括：

  • Claude Code（Anthropic）
  • Codex（OpenAI）
  • Gemini CLI（Google）
  • OpenCode（开源多模型 Agent）
  • OpenClaw（你的 OpenClaw！）

核心功能

功能模块 说明
Provider Management 一键切换不同 AI Provider 配置，无需手动修改环境变量或配置文件
MCP Server 管理 统一管理多个 MCP 服务器，支持 stdio / HTTP / SSE 三种传输类型，跨应用共享
Prompts 管理 多预设系统提示词，支持 Claude (CLAUDE.md)、Codex (AGENTS.md)、Gemini (GEMINI.md)
Skills 扩展管理 管理和快速切换 Skills 扩展配置
多端支持 桌面应用（Windows/macOS/Linux）+ Web 版本（无头服务器/SSH 环境）+ CLI 交互模式

安装方式

# macOS（推荐 Homebrew）
brew tap farion1231/ccswitch
brew install --cask cc-switch

# Windows
# 下载 .msi 安装包或便携版 .zip
# https://github.com/farion1231/cc-switch/releases

# Linux
# 下载 .deb 或 .AppImage
sudo dpkg -i cc-switch_*.deb

# Web 版本（无头服务器 / SSH 环境）
wget https://github.com/farion1231/cc-switch/releases/latest/download/cc-switch-web-linux-x64.tar.gz
tar -xzf cc-switch-web-linux-x64.tar.gz
cd cc-switch-web/
./cc-switch-web
# 浏览器访问 http://localhost:17666

🇨🇳 国产大模型配置指南

通过 CCSwitch 的 Provider Management 功能，可以将 MiniMax、Kimi（Moonshot）、Qwen（通义千问）、DeepSeek 等国产大模型配置为 Claude Code、Codex、OpenCode 的后端，实现一行命令切换国产大模型。

支持的国产大模型

模型 API 类型 特点 推荐场景
MiniMax（海螺） OpenAI 兼容 速度快、价格低、支持 Function Calling 日常编码、heartbeat、批量任务
Kimi（Moonshot） OpenAI 兼容 长上下文（128K/1M）、中文优化 大项目分析、长文档处理
Qwen（通义千问） OpenAI 兼容 + DashScope 阿里云原生、中文强大、开源版本可选 阿里云集成场景
DeepSeek OpenAI 兼容 性价比高、V3/R1 系列表现优秀 主力编码模型（成本敏感）
SiliconFlow（聚合 API） OpenAI 兼容 聚合多家模型、统一账单 需要灵活切换模型

在 CCSwitch 中配置国产大模型（Provider 配置）

1打开 CCSwitch → Provider Management

选择目标应用（Claude Code / Codex / OpenCode / OpenClaw）

2添加新 Provider 配置

填写配置名称（如 "MiniMax-编码"）和 API Endpoint（以 MiniMax 为例）：

# MiniMax API 端点（OpenAI 兼容格式）
Base URL: https://api.minimax.chat/v1

# 或通过 OpenRouter 等中转（兼容性更好）
Base URL: https://openrouter.ai/api/v1

# API Key: 你的密钥
# 模型: minimax/MiniMax-Text-01 或具体模型名
3配置四层模型层级

CCSwitch 支持配置 Haiku / Sonnet / Opus / Custom 四层模型，每个层级可以独立指定：

Main Model: minimax/MiniMax-Text-01    # 主模型
Haiku Model: minimax/MiniMax-Haiku      # 轻量快速
Sonnet Model: deepseek/deepseek-chat   # 均衡
Opus Model: openai/gpt-4o              # 最强
4速度测试 & 切换

CCSwitch 提供 API 延迟测试功能，测试后一键切换当前 Provider。切换后所有应用立即生效，无需重启。

使用 cc-switch CLI 快速切换

# 查看所有可用 Provider
cc-switch list

# 切换到指定 Provider
cc-switch use minimax-production

# 切换到上一个配置
cc-switch use --previous
cc-switch use -p

# 切换后自动启动 Claude Code
cc-switch use minimax-dev --launch

# 查看当前活跃的 Provider
cc-switch current
💡 OpenClaw + 国产大模型

☁️ Cloud Code（Claude Code）安装与 VS Code 集成

Claude Code 是 Anthropic 出品的 AI 编程 Agent，业界最强大的编程工具之一。它可以深度读写代码、执行 Terminal 命令、搜索文件、调用工具，并支持 Skills/Agents/Hooks 扩展系统。

🛠️ Claude Code Skills 详解

Skills 是 Claude Code 的核心扩展单元——每个 Skill 是一个自包含的文件夹，内含 SKILL.md 指令文件以及资源文件（脚本、模板、参考文档）。当用户任务匹配 Skill 的描述时，Claude 会自动加载对应知识，不需要手动调用。

两种 Skill 触发方式

类型 触发方式 示例
用户触发型 用户在对话中明确提及或调用 /plan "添加用户注册功能"
自动触发型 Claude 检测到任务匹配 Skill 描述时自动激活 写博客文章时自动调用 brand-voice-check Skill

Skill 文件结构

your-project/.claude/skills/
├── quick-audit/
│   ├── SKILL.md       # 核心指令（必须）
│   └── REFERENCE.md   # 参考文档（可选）
└── brand-voice-check/
    ├── SKILL.md
    ├── FORMS.md       # 表单模板
    └── scripts/
        └── check.sh   # 执行脚本

⚡ Commands / Hooks / Agents 工作流

Commands（斜杠命令）

Commands 是预定义的斜杠命令，在对话界面中输入 /command-name 即可触发特定工作流，通常用于需要固定流程的重复性任务。

# 常用内置 Commands
/claude --help          # 帮助
/claude --resume        # 恢复上一个会话
/model sonnet          # 切换到 Sonnet 模型

# 常见第三方 Commands（来自 everything-claude-code）
/tdd                   # 启动 TDD 工作流（红→绿→重构）
/code-review           # 代码审查
/ecc                   # ECC 全局命令入口
/ecc:plan "任务描述"    # 规划任务分解

Hooks（事件钩子）

Hooks 是事件驱动的自动化脚本——当特定事件发生时（如用户发送消息、Claude 执行工具、提交代码），Hook 自动执行，无需人工干预。这是保证代码质量的最有效手段。

支持的 Hook 事件类型

Hook 事件 触发时机 典型用途
BeforeMessage 用户发送消息之前 敏感词过滤、输入预处理
AfterMessage Claude 回复之后 记录日志、通知外部服务
ToolUse Claude 执行工具之前 拦截危险操作（如删除文件）
MatchTool 匹配特定工具调用 拦截非授权的 Bash/Write 操作
Edit 文件编辑前 保护特定文件（如密钥文件）

配置 Hooks（settings.json）

{
  "hooks": {
    "MatchTool": {
      "conditions": {
        "name": "Bash",
        "command": "rm -rf"
      },
      "action": "deny",
      "message": "危险操作：禁止使用 rm -rf 删除目录"
    },
    "Edit": {
      "conditions": {
        "filePath": "**/.env"
      },
      "action": "deny",
      "message": "禁止直接编辑 .env 文件"
    }
  }
}

自定义 Hook 脚本

# ~/.claude/hooks/block-dangerous.sh
#!/bin/bash
# 阻止危险的文件操作

TOOL_NAME=$(echo "$1" | jq -r '.name')
COMMAND=$(echo "$1" | jq -r '.command // ""')
FILE_PATH=$(echo "$1" | jq -r '.file_path // ""')

# 拦截 rm -rf /
if [[ "$COMMAND" == *"rm -rf"* ]] && [[ "$COMMAND" == *"/"* ]]; then
    echo "BLOCKED: Dangerous rm command detected"
    exit 1
fi

# 拦截删除 /system 路径
if [[ "$COMMAND" == *"rm"* ]] && [[ "$FILE_PATH" == /system/* ]]; then
    echo "BLOCKED: Cannot delete system paths"
    exit 1
fi

exit 0

Agents（子智能体）

Agents 是专门化的子智能体，擅长处理特定类型任务（如需求分析、TDD 红阶段、代码审查）。Claude Code 可以并行调用多个 Agent，形成流水线工作流。

常见的 Agent 类型：

  • TDD Red Agent——只写失败的测试（不写实现）
  • TDD Green Agent——实现功能让测试通过
  • TDD Refactor Agent——只做代码重构
  • Code Review Agent——安全性和质量审查
  • Task Planner Agent——任务拆解和计划生成

🔗 MCP（Model Context Protocol）扩展

MCP（Model Context Protocol）是 Anthropic 推出的开放协议，让 AI 工具可以调用外部数据源和工具。CCSwitch 内置 MCP Server 管理，支持 stdio / HTTP / SSE 三种传输方式。

热门 MCP 服务器推荐

MCP Server 用途 安装难度
context7 获取最新代码文档（替代过时的知识） 简单
filesystem 安全的文件读写操作 简单
GitHub Issues/PR/Repo 操作 中等
Jira 项目管理集成 中等
Playwright Web UI 自动化测试 中等
Database (PostgreSQL) 数据库查询和操作 较难

在 CCSwitch 中配置 MCP

# 1. 打开 CCSwitch → MCP Server Management
# 2. 点击 "Add MCP Server"
# 3. 选择传输类型：
#    - stdio: 本地进程通信（推荐）
#    - HTTP: 远程 MCP 服务
#    - SSE: Server-Sent Events

# 本地 stdio 示例（context7）
{
  "name": "context7",
  "transport": "stdio",
  "command": "npx",
  "args": ["-y", "@context7/mcp-server"],
  "description": "获取最新库文档"
}

# HTTP 远程 MCP 示例
{
  "name": "github-remote",
  "transport": "http",
  "url": "https://your-mcp-server.com/sse",
  "description": "远程 GitHub MCP"
}
