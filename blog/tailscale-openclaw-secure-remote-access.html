🔍 背景与问题

我们的 OpenClaw 跑在阿里云 ECS 上，通过钉钉与企业对接。之前有一个痛点：如何安全地从外部访问 ECS 上的 OpenClaw Dashboard？

阿里云 ECS 的安全组策略是只开放少量入方向端口，全部开放太危险。但每次手动加规则又很麻烦。理想的场景是：

  • 开放少量必要端口（如 SSH）
  • 不需要开放任何入站端口，就能从外网安全访问 Dashboard
  • 点对点加密通信，不暴露在公网
⚠️ 遇到的问题

同时还遇到了一个奇怪的问题：只要 Tailscale 开启，OpenClaw 就完全无法回复钉钉消息——消息能收到但发不出去超时。

📊 方案选型

方案 复杂度 安全性 备注
DDNS + 反向代理 高 中 需要域名、配置 Nginx、SSL 证书
开放全端口 无 危险 公网暴露，不可接受
端口映射（socat/frp） 中 中 需要中转服务器，延迟高
Tailscale 低 高 WireGuard 加密，P2P 直连，无需公网端口

最终选择 Tailscale，原因是：

  • 零配置 —— 安装后几分钟就能连通
  • 无公网端口暴露 —— 不需要在安全组开放任何入站规则
  • 端到端加密 —— WireGuard 协议，P2P 直连
  • 免费 —— 最多支持 100 台设备

🐛 网络问题排查：Tailscale 为何导致 OpenClaw 失联？

现象

Tailscale 开启后：

  • ❌ curl https://api.minimax.chat 返回 000（连接失败）
  • ❌ 钉钉消息收到但无法回复（发送超时）
  • ❌ Dashboard 也不响应

关掉 Tailscale 后立即恢复正常。

根因分析

ECS 的 DNS 原本由阿里云 VPC 提供（100.100.2.136 和 100.100.2.138），系统 DNS 指向 127.0.0.53（systemd-resolved）。

Tailscale 启动后，会接管 DNS 解析（--accept-dns=true 默认），将 ECS 的 DNS 请求转发到 Tailscale 提供的 DNS 服务器。而这个 DNS 服务器解析外网域名（如 api.minimax.chat）可能有限制或路由异常，导致所有外部 API 请求全部超时。

🔍 关键判断依据

查看 Tailscale 开启时的 DNS 配置：

# Tailscale 开启时的 DNS 配置
$ cat /etc/resolv.conf
nameserver 100.100.2.136   # ← Tailscale DNS，解析不了外网域名

# Tailscale 路由表
$ ip route | grep 100.
100.100.2.136 via 172.21.191.253 dev eth0  # Tailscale DNS 服务器路由
100.100.2.138 via 172.21.191.253 dev eth0

解决方案

最简单的方法：把 DNS 换成公共 DNS（Cloudflare 1.1.1.1 或 Google 8.8.8.8）：

echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf
⚠️ 注意

该修改在系统重启后可能被 systemd-resolved 覆盖，需要额外持久化配置（见后文"持久化 DNS 配置"章节）。

📝 完整操作步骤

第一步：注册 Tailscale

  • 访问 https://login.tailscale.com/register
  • 使用 Google 或 GitHub 账号登录
  • 免费账号支持最多 100 台设备

第二步：ECS 上安装 Tailscale

# 下载安装
curl -fsSL https://tailscale.com/install.sh | sh

# 启动服务
sudo systemctl unmask tailscaled
sudo systemctl start tailscaled

# 验证运行状态
sudo systemctl status tailscaled | head -5

第三步：生成 Auth Key 并连接

💡 为什么不用交互式登录？

因为 ECS 没有浏览器，无法完成 OAuth 回调。用 Auth Key 方式最稳定。

  1. 在你自己的电脑浏览器打开：
    https://login.tailscale.com/admin/settings/keys
  2. 点击 "Generate auth key"，创建设备认证 Key：
    • 给 Key 起个名字（如 "ECS server"）
    • 建议取消勾选"One-off"（否则用一次就失效）
    • 点击确认，复制生成的 tskey-auth-k... 开头的 Key
  3. 在 ECS 上执行（用你自己的 Key 替换）：
    sudo tailscale up --authkey=tskey-auth-kXXXXXXXXXXXXXXXXXXXXXXX
  4. 验证连接：
    sudo tailscale status

    应该能看到 ECS 和你的设备都在线，各自有一个 100.x.x.x 的 IP。

第四步：配置 DNS（解决网络冲突）

在 ECS 上运行：

# 临时生效
echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf

# 验证外网连通
curl -s --max-time 5 "https://api.minimax.chat" -o /dev/null -w "%{http_code}"
# 期望返回 200 或 308

第五步：持久化 DNS 配置

为了防止 systemd-resolved 在重启后覆盖 DNS 配置，创建 override 文件：

sudo mkdir -p /etc/systemd/resolved.conf.d
echo -e "[Resolve]
DNS=8.8.8.8
Domains=~" | sudo tee /etc/systemd/resolved.conf.d/use-google-dns.conf
sudo systemctl restart systemd-resolved

然后确认：

cat /etc/resolv.conf
# 应该显示 nameserver 8.8.8.8

🌐 如何从外网访问 ECS 上的服务

Tailscale 连接后，每台设备会分配一个 100.x.x.x 的虚拟 IP，可以通过这个 IP 直接访问 ECS 上的服务：

服务 访问方式
OpenClaw Dashboard 浏览器打开 http://100.x.x.x:18789
SSH ssh user@100.x.x.x
其他端口 直接 http://100.x.x.x:端口
💡 注意

安全组不需要开放任何入站端口，所有流量都通过 Tailscale 加密隧道。

❓ 常见问题

Q: 重启后 Tailscale 没有自动连接？

# 设置开机自启
sudo systemctl enable tailscaled
sudo systemctl start tailscaled

Q: Auth Key 过期了怎么办？

去 Tailscale 管理后台重新生成一个新的 Key，然后在 ECS 上运行：

sudo tailscale up --authkey=新key

Q: 如何确认 Tailscale 连接是否正常？

sudo tailscale status
ping -c 2 100.x.x.x  # 用对方设备的 Tailscale IP

✅ 总结

通过 Tailscale，我们实现了：

  • ✅ 零公网端口暴露 —— 安全组无需额外开放入站规则
  • ✅ 端到端加密 —— WireGuard P2P 直连
  • ✅ 快速部署 —— 全程约 10 分钟
  • ✅ 多设备互通 —— 手机、电脑、ECS 组成虚拟局域网
  • ✅ Dashboard 远程访问 —— 通过 Tailscale IP 随时访问

唯一要注意的是 DNS 配置冲突问题，解决方法就是使用公共 DNS（8.8.8.8）。
