🔍 背景：为什么放弃 Tailscale？

之前我尝试使用 Tailscale 来实现服务器远程访问，但遇到了严重问题：

  • 钉钉和飞书的 WebSocket 连接频繁断开
  • DNS 解析被劫持到 Google 等国外服务器
  • 国内应用无法正常使用
  • 服务不稳定，依赖国外服务器

这让我不得不深入研究 Tailscale 的底层原理，以及寻找更适合国内的替代方案。

⚠️ Tailscale 的核心问题

Tailscale 是一个商业产品，虽然基于 WireGuard 协议，但它依赖 Tailscale 官方的协调服务器（Coordination Server）来帮助设备建立连接。这个服务器在国外，导致国内用户体验很差。

⚖️ WireGuard vs Tailscale：核心区别

很多人在讨论"WireGuard 和 Tailscale 哪个好"，但实际上它们不是同一个层面的东西：

对比项 WireGuard Tailscale
性质 VPN 协议/技术标准 商业产品/应用
协议 WireGuard 协议（开源） 基于 WireGuard + 私有协调
服务器 自建，完全可控 依赖 Tailscale 官方服务器
DNS 完全自控，不会劫持 会修改系统 DNS 设置
易用性 需要手动配置 安装即用，适合小白
成本 只有服务器成本 免费版够用，付费更强大
国内可用性 ✅ 完全可用 ⚠️ 依赖国外服务器，有风险
🔑 关键理解

WireGuard 是一个 VPN 协议，就像 HTTP 是 web 协议一样。它是开源标准，任何人都可以免费使用。

Tailscale 是一个商业产品，它用 WireGuard 协议作为底层，但加入了用户体验层（自动 NAT 穿透、魔法链接、流量路由等），代价是需要连接到它的协调服务器。

🔧 WireGuard 工作原理

1. 密钥对机制

WireGuard 使用非对称加密，每个设备有一对密钥：

  • 私钥：保管在本地，永不泄露
  • 公钥：分享给需要通信的其他设备

密钥对使用 Curve25519 算法，安全强度极高。

2. 隧道接口

WireGuard 在系统中创建一个虚拟网卡（如 wg0），分配一个内网 IP（如 10.0.0.2）：

wg0: flags=209
    inet 10.0.0.2  netmask 255.255.255.0
    peer: 10.0.0.1  endpoint 123.456.789.10:51820

所有发往这个网段的流量都会通过 WireGuard 隧道传输。

3. 加密隧道建立过程

WireGuard 的连接建立过程简洁高效：

  1. 交换公钥：两端互相交换公钥（通过任何渠道）
  2. 数据包包装：发送方用对方公钥加密数据包
  3. 隧道传输：加密数据包通过公网发送到对方
  4. 数据包解包：接收方用私钥解密，拿到原始数据

整个过程在 Linux 内核中完成，性能极高。

4. NAT 穿透

WireGuard 协议设计优雅，支持 UDP NAT 穿透：

  • 同一局域网：设备直接通信，速度最快
  • 对称 NAT：需要 UDP 打洞，碰壁成功率约 80%
  • 无法穿透：极少数情况需要中继服务器

与 Tailscale 不同的是：WireGuard 没有"协调服务器"的概念，纯 P2P。

🌐 为什么 Tailscale 会搞乱 DNS？

Tailscale 有一个争议性很大的功能：MagicDNS。

当 Tailscale 连接时，它会：

  1. 接管系统的 DNS 设置（修改 /etc/resolv.conf）
  2. 让 DNS 查询优先走 Tailscale 的 DNS 服务器
  3. 这个 DNS 服务器会将 *.tailnet.net 域名解析到 Tailscale 内网 IP
  4. 但它也会转发其他 DNS 查询到境外服务器
⚠️ DNS 劫持的后果
  • 钉钉/飞书的 WebSocket 连接指向的 IP 发生变化
  • 连接目标 IP 变成境外 IP 或无法访问
  • 应用认为连接失败，断开 WebSocket
  • 轻则消息延迟，重则完全掉线

Tailscale 的"魔法"代价

Tailscale 为了用户体验，做了很多"魔法"操作：

  • MagicDNS：简化网络中的设备命名
  • GlobalNAT：帮助穿透对称 NAT
  • Funnel：公开暴露本地服务
  • SSH 穿越：通过 Tailscale 网络 SSH

这些功能都需要 Tailscale 官方服务器参与，增加了复杂性和隐私风险。

🏗️ 自建 WireGuard 方案

自建 WireGuard VPN 的优势：

  • ✅ 完全自主：服务器在国内，延迟低
  • ✅ 不劫持 DNS：不会影响钉钉/飞书
  • ✅ 流量可控：不走任何境外服务器
  • ✅ 成本低廉：只需一台低配服务器
  • ✅ 简单可靠：协议简洁，稳定性极高
📊 推荐架构 
Windows 电脑 (WireGuard Client)
       ↓ UDP 包
  国内云服务器 (WireGuard Server)
       ↓
  访问 127.0.0.1:18789 (OpenClaw Dashboard)
  访问 127.0.0.1:22 (SSH)

📦 安装配置详解

1. 服务端安装（Ubuntu）

# 安装 WireGuard
sudo apt update
sudo apt install wireguard -y

# 安装 qrencode（用于生成客户端二维码）
sudo apt install qrencode -y

2. 生成密钥对

# 服务端密钥
wg genkey | tee server_private.key | wg pubkey > server_public.key

# 客户端密钥（在每台设备上生成）
wg genkey | tee client_private.key | wg pubkey > client_public.key

3. 服务端配置

# 创建配置文件
sudo nano /etc/wireguard/wg0.conf

写入以下内容：

[Interface]
# 服务端私钥
PrivateKey = <服务端私钥>

# WireGuard 网卡 IP
Address = 10.0.0.1/24

# 监听端口
ListenPort = 51820

# 防火墙规则
PostUp = iptables -A FORWARD -i wg0 -j ACCEPT; iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
PostDown = iptables -D FORWARD -i wg0 -j ACCEPT; iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE

# 客户端配置
[Peer]
# 客户端公钥
PublicKey = <客户端公钥>

# 客户端内网 IP
AllowedIPs = 10.0.0.2/32

4. 启用 IP 转发

# 开启 IP 转发
echo "net.ipv4.ip_forward = 1" | sudo tee -a /etc/sysctl.conf
echo "net.ipv6.conf.all.forwarding = 1" | sudo tee -a /etc/sysctl.conf
sudo sysctl -p

5. 启动 WireGuard

# 启动服务
sudo systemctl start wg-quick@wg0

# 设置开机自启
sudo systemctl enable wg-quick@wg0

# 查看状态
sudo wg show

6. 阿里云安全组配置

在阿里云控制台开放 UDP 51820 端口：

  • 协议：UDP
  • 端口范围：51820/51820
  • 来源：0.0.0.0/0（或指定 IP）

7. 客户端配置（Windows）

Step 1：下载 WireGuard 客户端

访问 WireGuard 官方下载页面，下载 Windows 安装包并安装。

Step 2：生成本地密钥对

在 Windows 上打开 PowerShell，运行以下命令生成密钥对：

# 生成本地私钥
$privateKey = wg genkey

# 生成公钥
$publicKey = $privateKey | wg pubkey

# 将私钥和公钥保存到文件（方便后续使用）
$privateKey | Out-File -FilePath "$env:USERPROFILE\wg_private.key"
$publicKey | Out-File -FilePath "$env:USERPROFILE\wg_public.key"

# 显示公钥（需要复制给服务端）
Write-Host "你的公钥是:"
Get-Content "$env:USERPROFILE\wg_public.key"

将生成的公钥复制下来，稍后填入服务端的 wg0.conf 配置中。

Step 3：创建客户端配置文件

在 Windows 上打开记事本，创建以下配置文件，保存到 C:\Program Files\WireGuard\Config\wg0.conf：

[Interface]
# Windows 本地私钥
PrivateKey = <客户端私钥（从 PowerShell 生成的私钥）>

# 客户端内网 IP
Address = 10.0.0.2/24

# 国内首选 DNS（避免 DNS 劫持）
DNS = 223.5.5.5, 119.29.29.29

[Peer]
# 服务端公钥
PublicKey = <服务端公钥>

# 服务器地址和端口
Endpoint = your-server-ip:51820

# 流量走向：0.0.0.0/0 = 全局 VPN（推荐）；10.0.0.0/24 = 仅访问内网
AllowedIPs = 0.0.0.0/0

# 保持连接（穿透 NAT）
PersistentKeepalive = 25
📝 配置说明
  • PrivateKey：填入刚才 PowerShell 生成的私钥
  • PublicKey：填入服务端的公钥
  • Endpoint：填入你的服务器 IP
  • AllowedIPs：0.0.0.0/0 = 全局翻转到服务器；10.0.0.0/24 = 仅访问内网，不影响其他流量

Step 4：导入配置并连接

  • 打开 WireGuard 客户端
  • 点击"导入隧道"按钮，选择刚才创建的 wg0.conf 文件
  • 选中导入的隧道，点击"激活"按钮
  • 任务栏图标变绿表示连接成功

8. 验证连接

连接成功后，在 Windows 上打开 PowerShell，验证是否能访问服务器内网：

# 测试能否 ping 通服务器端
ping 10.0.0.1

# 查看 WireGuard 状态
wg show

# 查看当前分配的 IP
ipconfig

如果一切正常，你应该能看到 wg0 虚拟网卡获取到了 10.0.0.2 的 IP。

🇨🇳 国内可用替代方案

1. ZeroTier

类似 Tailscale 的商业方案，但：

  • ✅ 支持自建 Planet 服务器（moon/planet）
  • ✅ 有国内用户基础
  • ⚠️ 默认仍依赖国外根服务器
  • ⚠️ 自建需要额外配置

2. Headscale

Tailscale 控制服务器的开源实现：

  • ✅ 完全开源，自建控制服务器
  • ✅ 兼容 Tailscale 客户端
  • ✅ 不依赖 Tailscale 官方
  • ❌ 配置复杂，需要一定技术基础

3. n2n

老牌 P2P VPN 方案：

  • ✅ 完全开源，P2P 直连
  • ✅ 支持边缘节点中继
  • ✅ 国内有使用群体
  • ❌ 文档较少，配置较复杂

📝 总结与建议

什么时候用 Tailscale？

  • ✅ 在国外使用
  • ✅ 对隐私要求不高
  • ✅ 需要快速上手，不想折腾

什么时候自建 WireGuard？

  • ✅ 在国内使用
  • ✅ 对稳定性要求高
  • ✅ 不希望 DNS/流量被劫持
  • ✅ 技术能力足够
  • ✅ 使用钉钉/飞书等国内应用
💡 最终建议

如果只是 SSH + OpenClaw Dashboard 远程访问，现在的方案已经足够安全：

  • SSH 密钥登录 + 只开放 SSH 端口
  • Dashboard 只监听 127.0.0.1
  • 不需要任何额外 VPN 软件

如果需要多设备组网（手机、平板、电脑同时访问内网），再考虑 WireGuard 自建。
