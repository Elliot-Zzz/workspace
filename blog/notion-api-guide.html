📖 Notion API 对接指南：用免费版 Notion 打造你的自动化知识库

标签：工具效率

很多人在用 Notion 免费版时会担心：API 功能是不是付费版才有的？实际上，Notion 的 API 对所有用户免费开放，无论你是免费版、个人版还是团队版，只要创建了 Integration，就能通过 API 读写 Notion 的内容。

本文详细介绍如何将 Notion 与 OpenClaw 对接，实现文档管理、知识库自动化、任务追踪等场景。

一、Notion API 核心概念

1.1 什么是 Integration？

Integration 是 Notion 对外开放 API 的"身份凭证"。创建后，Integration 相当于一个机器人账号，可以读取和写入被授权的页面和数据库。

  • API Key 格式：ntn_ 或 secret_ 开头
  • API 版本：目前使用 2025-09-03（最新）
  • 创建地址：https://notion.so/my-integrations

1.2 免费版与付费版的区别

功能免费版付费版
API 读写页面✅ 支持✅ 支持
API 读写数据库✅ 支持✅ 支持
API 请求频率限制~3次/秒~3次/秒
页面版本历史❌ 无✅ 有
实时协作人数10人无限

结论：API 能力完全一样，付费版主要差在协作人数和高级分析功能。

二、五分钟快速上手

步骤 1：创建 Notion Integration

  1. 打开 https://notion.so/my-integrations
  2. 点击 "New integration"
  3. 填写名称（如"OpenClaw知识库"）
  4. 选择关联的 Workspace
  5. 点击 Submit，复制生成的 API Key

步骤 2：配置到本地

# 创建配置目录
mkdir -p ~/.config/notion

# 写入 API Key
echo "ntn_your_key_here" > ~/.config/notion/api_key

步骤 3：授权 Integration 访问页面

这是免费版用户最容易卡住的地方：

  1. 打开你要授权的 Notion 页面
  2. 点击右上角 "..."（更多操作）
  3. 选择 "Connections" → "Connect to"
  4. 选择你创建的 Integration 名称
⚠️ 重要：没有被授权的页面，API 会报 401 Unauthorized 错误。

步骤 4：验证连接

NOTION_KEY=$(cat ~/.config/notion/api_key)
curl -X POST "https://api.notion.com/v1/search" \
  -H "Authorization: Bearer $NOTION_KEY" \
  -H "Notion-Version: 2025-09-03" \
  -H "Content-Type: application/json" \
  -d '{"query": "test"}'

返回 200 即连接成功。

三、OpenClaw Notion Skill 常用命令

3.1 搜索页面和数据库

curl -X POST "https://api.notion.com/v1/search" \
  -H "Authorization: Bearer $NOTION_KEY" \
  -H "Notion-Version: 2025-09-03" \
  -H "Content-Type: application/json" \
  -d '{"query": "项目文档"}'

3.2 读取页面内容

# 获取页面基本信息
curl "https://api.notion.com/v1/pages/{page_id}" \
  -H "Authorization: Bearer $NOTION_KEY" \
  -H "Notion-Version: 2025-09-03"

# 获取页面正文内容（blocks）
curl "https://api.notion.com/v1/blocks/{page_id}/children" \
  -H "Authorization: Bearer $NOTION_KEY" \
  -H "Notion-Version: 2025-09-03"

3.3 创建新页面

curl -X POST "https://api.notion.com/v1/pages" \
  -H "Authorization: Bearer $NOTION_KEY" \
  -H "Notion-Version: 2025-09-03" \
  -H "Content-Type: application/json" \
  -d '{
    "parent": {"database_id": "你的数据库ID"},
    "properties": {
      "Name": {"title": [{"text": {"content": "新任务"}}]},
      "Status": {"select": {"name": "Todo"}}
    }
  }'

3.4 查询数据库

curl -X POST "https://api.notion.com/v1/data_sources/{data_source_id}/query" \
  -H "Authorization: Bearer $NOTION_KEY" \
  -H "Notion-Version: 2025-09-03" \
  -H "Content-Type: application/json" \
  -d '{
    "filter": {"property": "Status", "select": {"equals": "Active"}},
    "sorts": [{"property": "Date", "direction": "descending"}]
  }'

四、实战场景举例

场景 1：会议纪要自动归档

每次会议结束后，OpenClaw 自动将纪要写入 Notion 的"会议记录"数据库，包含时间、参会人、决议事项。

场景 2：项目任务追踪

在 Notion 数据库中管理任务状态，OpenClaw 定时查询"待处理"任务，通过钉钉推送给相关人员。

场景 3：知识库同步

将 OpenClaw 处理过的文档、技术方案自动同步到 Notion，形成统一的知识沉淀。

五、注意事项

  1. 频率限制：API 平均约 3 次/秒，收到 429 rate_limited 时根据 Retry-After 头等待后重试
  2. Payload 限制：每次最多 1000 个 block 元素，总大小 500KB
  3. 权限必须手动授权：私人页面不开放给 Integration，API 读不到
  4. 数据库改名为 Data Source：API 端点在 2025-09-03 版本中已将"数据库"改称"数据源"，但创建页面时仍用 database_id

六、总结

Notion 免费版完全支持 API 对接，能力与付费版无异。配合 OpenClaw 的 notion skill，可以轻松实现：

  • 📝 文档自动归档
  • ✅ 任务状态追踪
  • 🔍 知识库检索
  • 📊 数据统计分析

只需一个免费的 API Key，5 分钟就能跑通第一个场景。
