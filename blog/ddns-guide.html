🔍 背景：无公网IP的困境

国内大多数家庭宽带用户都面临一个问题：没有公网IP。

运营商通常提供的是：

  • NAT 私家基站：手机信号基站，共享公网出口
  • 大内网：整栋楼或小区共享一个公网IP

这意味着：

  • ❌ 无法直接从外部访问家庭服务器
  • ❌ 无法绑定域名做 HTTPS
  • ❌ SSH、远程桌面都很麻烦
⚠️ 常见误区

很多人以为有了域名就能访问，这是错的。域名只是门牌号，它需要指向一个能被外部访问的IP。如果IP是内网的，域名毫无意义。

⚖️ 主流方案对比

解决无公网IP访问内网的方案有以下几种：

方案 原理 速度 配置难度 成本
DDNS + 端口映射 动态域名指向家宽IP，路由器端口映射 🚀 快（直连） ⭐⭐ 简单 免费~¥30/年
内网穿透（frp/nat123） 中继服务器转发流量 🐢 取决于中继 ⭐⭐⭐ 中等 免费~¥50/月
Tailscale/ZeroTier P2P VPN 组网 🚀 快（P2P） ⭐ 简单 免费
向日葵/ToDesk 商业远程桌面软件 🐢 一般 ⭐ 简单 免费~¥200/年
💡 本文重点：DDNS + 端口映射

这是速度最快、成本最低的方案。路由器拨号获取真实公网IP时完全可行。

🔧 DDNS 工作原理

什么是 DDNS？

DDNS（Dynamic Domain Name System） = 动态域名系统。

普通 DNS 是"静态的"：域名 → 固定IP，IP 变了就失效。

DDNS 是"动态的"：域名 → 随时变化的IP，IP 变了自动更新。

DDNS 工作流程

┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│  家庭路由器  │      │  DDNS 服务商  │      │   你的电脑   │
│  (拨号获取IP)│      │   (云端)      │      │   (在外面)   │
└──────┬──────┘      └──────┬──────┘      └──────┬──────┘
       │                    │                    │
       │  1. 检测到IP变化    │                    │
       │───────────────────>│                    │
       │                    │                    │
       │  2. 更新域名解析     │                    │
       │                    │                    │
       │<────────────────────────────────────────│
       │    3. 域名 → 你的公网IP                  │
       │────────────────────────────────────────>│
       │                    │                    │
       │  4. 访问家庭服务器                         │
       │<────────────────────────────────────────│
       │                    │                    │
  1. 路由器检测到自己的公网IP发生变化
  2. 路由器把新IP发送给 DDNS 服务商
  3. DDNS 服务商更新域名解析
  4. 外部设备通过域名访问，自动路由到你的家庭服务器

关键前提：需要真实的公网IP

DDNS + 端口映射的前提是：你的路由器能获取到真实的公网IP。

如何判断：

  • 登录路由器管理界面，查看 WAN 口 IP
  • 访问 IP138，对比两个IP是否一致
  • 如果一致 → 可以用 DDNS；如果不一致（是内网IP）→ 需要其他方案

📦 路由器 DDNS 配置（推荐）

Step 1：确认路由器支持 DDNS

大多数中高端路由器（小米、华为、华硕、TP-Link）都内置 DDNS 功能。

登录路由器后台，查找：

  • 高级设置 → 外部管理 → DDNS
  • 路由设置 → 动态DNS

Step 2：注册 DDNS 服务商

国内推荐使用 花生壳（免费版够用）或腾讯云/阿里云解析。

以花生壳为例：

  1. 访问 花生壳官网 注册账号
  2. 登录后在"管理中心"创建"内网穿透"服务
  3. 会获得一个免费域名（如 xxx.oicp.net）
  4. 或者购买自己的域名（如 yourname.com）

Step 3：在路由器中配置 DDNS

登录路由器后台，找到 DDNS 设置页面，填写：

配置项 示例值 说明
服务商 花生壳 / Oray 选择你注册的服务商
域名 yourname.oicp.net 服务商提供的免费域名
用户名/密码 oray账号/密码 DDNS服务商的登录凭证
更新间隔 5 分钟 检测IP变化的频率

保存后，路由器会自动定时检测IP变化并更新。

Step 4：配置端口映射（端口转发）

DDNS 只是让域名指向你的IP，端口映射才是把外部请求送到内网服务器。

在路由器中找到"端口映射"或"虚拟服务器"：

配置项 示例值 说明
外部端口 22022 外网访问用的端口（建议改非标准端口）
内部IP 192.168.1.100 你服务器的局域网IP
内部端口 22 服务器上SSH的实际端口
协议 TCP 通常选TCP即可
✅ 配置完成后

外部访问流程：

你的电脑 ---> ssh -p 22022 user@yourname.oicp.net
                              ↓
                        路由器:22022
                              ↓
                        端口映射
                              ↓
                        服务器:22

🌐 免费 DDNS 服务商推荐

1. 花生壳（Oray）— 推荐国内用户

  • ✅ 免费版提供 *.oicp.net 域名
  • ✅ 路由器支持率高，内网穿透功能完善
  • ✅ 有客户端软件，手机也能管理
  • ⚠️ 免费版带宽有限（2Mbps），商业使用建议付费
  • ⚠️ 域名较丑，不适合绑定到自己的域名

2. 腾讯云解析 / 阿里云解析 — 自有域名首选

  • ✅ 域名完全自主，可绑SSL证书
  • ✅ 免费额度大（腾讯云100万次/日）
  • ✅ 适合有自己域名的用户
  • ❌ 需要自己写脚本更新IP（无路由器内置支持）
  • ❌ 首次配置稍复杂

如果你的域名在腾讯云/阿里云，可以用下面的脚本配合 DDNS：

# 需设置环境变量：TX_SECRET_ID, TX_SECRET_KEY, DOMAIN, RECORD_LINE

# 获取当前公网IP
CURRENT_IP=$(curl -s https://api.ipify.org)

# 调用腾讯云API更新解析记录
# 具体参数参考腾讯云文档

3. Dynu / No-IP — 国际老牌

  • ✅ 完全免费，域名如 *.dynu.com
  • ✅ API 支持完善
  • ⚠️ 服务器在国外，国内速度一般
  • ⚠️ 免费版需每月确认一次（防止滥用）

📝 完整配置流程（以花生壳为例）

第一步：注册花生壳并实名认证

  1. 访问 花生壳官网
  2. 注册账号（需要手机号）
  3. 完成实名认证（国内服务商必需）

第二步：添加内网映射

  1. 登录花生壳管理后台
  2. 点击"添加映射"
  3. 填写信息：
    • 应用名称：SSH访问
    • 内网主机：192.168.1.100（服务器IP）
    • 内网端口：22（SSH）
    • 外网端口：22022（自定义）
  4. 保存后会获得外网访问地址

第三步：测试访问

# 在外网电脑或手机上测试
ssh -p 22022 user@your-server

# 或直接访问内网穿透的地址
ssh -p 22022 user@xxx.oicp.net
🎉 花生壳内网穿透 vs DDNS 的区别

注意！花生壳的"内网穿透"和传统DDNS是两回事：

  • 内网穿透：通过花生壳服务器中继，不要求公网IP，但速度受限
  • DDNS + 端口映射：直连你的公网IP，速度快，但需要路由器有公网IP

如果你的宽带有公网IP，优先使用 DDNS + 端口映射！

🔐 安全注意事项

1. 修改SSH默认端口

22端口是扫描器重点目标，建议改成高位端口（如 22022）：

# 编辑 SSH 配置
sudo nano /etc/ssh/sshd_config

# 找到 Port 22，改为
Port 22022

# 重启SSH
sudo systemctl restart sshd

2. 使用密钥登录，禁止密码登录

# 生成SSH密钥对（本地电脑）
ssh-keygen -t ed25519

# 复制公钥到服务器
ssh-copy-id -p 22022 user@yourname.oicp.net

# 编辑SSH配置禁用密码登录
sudo nano /etc/ssh/sshd_config
# 添加或修改：
PasswordAuthentication no
PubkeyAuthentication yes

3. 使用fail2ban防止暴力破解

# 安装fail2ban
sudo apt install fail2ban -y

# 启动并设置开机自启
sudo systemctl enable fail2ban
sudo systemctl start fail2ban

4. 防火墙规则

# 只允许特定IP访问SSH（推荐）
sudo ufw allow from 你的固定IP to any port 22022

# 或只允许内网访问
sudo ufw allow from 192.168.1.0/24 to any port 22022
⚠️ 重要提醒
  • DDNS域名本身有被封风险，建议同时做数据备份
  • 不要把所有端口都暴露，按需开放
  • 定期检查路由器固件更新
  • 有条件的话使用 VPN（如 WireGuard）代替直接暴露端口

📝 总结

DDNS + 端口映射适合的场景

  • ✅ 家庭宽带有公网IP
  • ✅ 需要SSH、远程桌面访问
  • ✅ 想绑定自己的域名
  • ✅ 对速度要求高（直连无中继）

需要用其他方案的场景

  • ❌ 宽带没有公网IP → 用花生壳内网穿透或 ZeroTier
  • ❌ 需要HTTPS访问 → 用 Cloudflare Tunnel 或 frp + Nginx
  • ❌ 不想折腾 → 用向日葵/ToDesk
💡 最佳实践

DDNS + WireGuard 是完美的家庭远程访问组合：

  • DDNS：解决域名指向问题
  • WireGuard：提供加密隧道，无需暴露任何端口
  • 端口映射：仅在需要时使用
