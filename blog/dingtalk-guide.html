💬 OpenClaw 钉钉集成

dingtalk vs dingtalk-setup 区别与使用指南

引言

在使用 OpenClaw 连接钉钉时，你可能会看到两个相关的 Skill：dingtalk 和 dingtalk-setup。它们分别扮演什么角色？有什么区别？本文将为你详细解答。

1. 两个 Skill 的区别

特性 dingtalk dingtalk-setup
性质 OpenClaw 插件 (Channel Plugin) 配置引导 Skill
作用 实际处理钉钉消息的发送/接收 指导用户完成钉钉渠道的配置
功能 消息收发、用户管理、群聊支持 安装插件、配置参数、排查问题
是否必需 ✅ 必需（核心组件） ❌ 可选（仅首次配置时使用）
使用频率 持续运行 一次性

2. dingtalk 插件详解

2.1 是什么？

dingtalk 是 OpenClaw 的钉钉通道插件，负责：

  • ✅ 接收钉钉消息
  • ✅ 发送钉钉消息
  • ✅ 支持企业内部应用和 Webhook 机器人
  • ✅ 私聊 (DM) 和群聊管理
  • ✅ 用户白名单支持

2.2 配置方式

方式一：环境变量

export DINGTALK_CLIENT_ID="your-app-key"
export DINGTALK_CLIENT_SECRET="your-app-secret"

方式二：配置文件

openclaw config --section channels
# 选择 DingTalk 并按提示操作

方式三：Webhook 机器人（群聊）

{
  "channels": [{
    "id": "dingtalk",
    "plugin": "dingtalk",
    "enabled": true,
    "webhookUrl": "https://oapi.dingtalk.com/robot/send?access_token=xxxxx",
    "webhookSecret": "SECxxxxx"
  }]
}

2.3 完整配置参数

参数必需说明
clientId✅钉钉 App Key
clientSecret✅钉钉 App Secret
robotCode❌机器人代码
corpId❌企业 ID
agentId❌应用 Agent ID
dm❌私聊策略：open / allowlist
group❌群聊策略：open / allowlist

3. dingtalk-setup 详解

3.1 是什么？

dingtalk-setup 是一个配置引导 Skill，专门帮助用户：

  • 🤖 安装 dingtalk 插件
  • ⚙️ 配置钉钉渠道参数
  • 🔧 排查钉钉消息问题
  • 📝 提供自动化部署脚本

3.2 使用场景

✅ 应该使用：

  • 用户说"对接钉钉"、"配置钉钉机器人"
  • 钉钉消息没有响应，需要排查
  • 需要安装钉钉插件

❌ 不应该使用：

  • 飞书/Lark 配置 → 使用 feishu skill
  • 企业微信 → 使用 wecom skill
  • Telegram/Discord → 使用对应的 channel skill

3.3 配置步骤

Step 1: 安装插件

# 国内网络（推荐）
$env:NPM_CONFIG_REGISTRY="https://registry.npmmirror.com"
openclaw plugins install @soimy/dingtalk

# 国际网络
openclaw plugins install @soimy/dingtalk

Step 2: 启用插件

# 编辑 ~/.openclaw/openclaw.json
{
  "plugins": {
    "enabled": true,
    "allow": ["dingtalk"]
  }
}

Step 3: 配置渠道

# 交互式配置（推荐）
openclaw configure --section channels

# 或手动编辑配置文件
openclaw config --section channels

Step 4: 重启 Gateway

openclaw gateway restart

4. 常见问题排查

🔧 排查清单
  • ✅ Gateway 运行中 → openclaw gateway status
  • ✅ 插件在白名单 → 检查 plugins.allow 包含 "dingtalk"
  • ✅ 渠道已启用 → 检查 channels[].enabled 为 true
  • ✅ 凭证正确 → 在钉钉开放平台验证 Client ID/Secret
  • ✅ 重启 Gateway → openclaw gateway restart

常见错误

错误解决方案
Plugin not found确认插件已安装，检查目录存在
Connection failed验证凭证，检查网络连接
Plugin not loaded添加到 plugins.allow，重启 Gateway
invalid timestamp同步系统时间
app not authorized检查钉钉开放平台应用权限

5. 总结

简单理解两者的关系：

  • dingtalk = 司机（负责实际开车）
  • dingtalk-setup = 教练（教你如何开车，教会就走）

推荐流程：

  1. 首次配置：使用 dingtalk-setup 完成安装和配置
  2. 日常使用：dingtalk 插件自动处理消息
  3. 遇到问题：再次参考 dingtalk-setup 排查
