一、DWS 是什么？

DWS（ DingTalk Workspace CLI）是钉钉官方提供的命令行工具，通过统一的 dws 命令覆盖钉钉全线产品能力。与直接调用 HTTP API 不同，DWS 封装了认证、错误处理、数据解析等繁琐细节，运维人员只需关注业务逻辑。

核心优势：
  • ✅ 统一 CLI，体验一致，所有产品共用同一套 flag 规范（--format json、--yes 等）
  • ✅ JSON 输出 + --help + dws schema 三位一体文档，告别"参数靠猜"
  • ✅ 危险操作二次确认（--yes 显式授权）
  • ✅ 批量操作有脚本（scripts/ 目录），拿来即用
  • ✅ 单次批量不超过 30 条记录，服务端限流友好

版本要求

CLI 版本需 >= 1.0.15，通过 dws --version 确认。当前 OpenClaw 已集成 DWS，Elaine 可直接调用。

三、命令发现与文档规范

DWS 的权威文档有两层，发生冲突时按此优先级：dws schema > dws <cmd> --help > 参考文档（.md）。

3.1 人读视图 — 看 Usage 和 Example

# 查看命令帮助
dws <command-path> --help

# 示例：日历命令帮助
dws calendar event list --help

3.2 机读视图 — JSON Schema + Flag 别名

# 列出所有产品及工具
dws schema

# 查看特定命令的 Schema
dws schema calendar.list_suggested_event_times
dws schema "calendar event list"

# 只看 flag 别名（实际生效的参数名）
dws schema <path> --jq '.tool.flag_overlay'

# 只看必填字段
dws schema <path> --jq '.tool.required'
⚠️ 注意：所有命令必须加 --format json 获取可解析输出。危险操作（删除、审批等）需用户确认后才加 --yes 执行。

四、运维实战场景

场景1：故障告警通知 — Webhook 机器人

🤖 需求：服务器 CPU 超 90%，自动发钉钉群告警

通过 Webhook 机器人发送 Markdown 格式告警，可 @ 值班人员。

# 发送 Webhook 告警（无需认证，机器人 Code 不需要）
dws chat message send-by-webhook \
  --token <webhook-token> \
  --title "🚨 服务器告警" \
  --text "**服务器：**生产环境-1\n**指标：**CPU 92%\n**时间：**2026-04-30 13:00:00\n请值班人员及时处理！" \
  --at-users 1104083318783932 \
  --format json

适用命令chat message send-by-webhook

场景2：紧急故障必须响应 — DING 消息

📞 需求：P0 故障需立即通知，不接受忽略

DING 消息支持应用内、短信、电话三种方式，短信/电话可确保触达。

# 应用内 DING（免费）
dws ding message send \
  --robot-code <ROBOT_CODE> \
  --type app \
  --users <USER_ID> \
  --content "P0故障：数据库连接失败，请立即处理！" \
  --format json

# 电话 DING（紧急，有成本）
dws ding message send \
  --robot-code <ROBOT_CODE> \
  --type call \
  --users <USER_ID> \
  --content "紧急：核心服务不可用，请接听！" \
  --format json

⚠️ 危险 电话/短信 DING 有通信成本，使用前需确认。

适用命令ding message send

场景3：故障跟进任务派发 — Todo 待办

✅ 需求：故障发生后派发跟进任务，设置优先级和截止时间

# 创建紧急待办
dws todo task create \
  --title "【P0】修复数据库连接池泄漏" \
  --executors <USER_ID> \
  --priority 40 \
  --due "2026-04-30T18:00:00+08:00" \
  --format json

# 查询未完成待办
dws todo task list --page 1 --size 20 --status false --format json

# 标记完成
dws todo task done --task-id <taskId> --status true --format json

适用命令todo task create / list / done

场景4：团队值班排班 — Calendar 日历

📅 需求：创建值班日程，确认参与人时间冲突

# 创建值班日程
dws calendar event create \
  --title "周末值班" \
  --start "2026-05-01T09:00:00+08:00" \
  --end "2026-05-01T18:00:00+08:00" \
  --attendee-user-ids <USER_ID_1>,<USER_ID_2> \
  --format json

# 查询会议室闲忙
dws calendar room freebusy \
  --room-ids <ROOM_ID> \
  --start "2026-05-01T09:00:00+08:00" \
  --end "2026-05-01T18:00:00+08:00" \
  --format json

# 获取时间建议（找出大家都有空的时间）
dws calendar event suggest-times \
  --attendee-user-ids <USER_ID_1>,<USER_ID_2> \
  --duration 60 \
  --format json

适用命令calendar event create / room freebusy / event suggest-times

场景5：运维工单审批 — OA 审批

📋 需求：查询待处理工单，审批通过或驳回

# Step 1: 查看待我处理的审批
dws oa approval list-pending --page 1 --size 20 --format json
# → 返回 processInstanceId

# Step 2: 查看工单详情
dws oa approval detail --instance-id <processInstanceId> --format json

# Step 3: 获取 taskId（审批动作必须用 taskId）
dws oa approval tasks --instance-id <processInstanceId> --format json

# Step 4: 同意审批
dws oa approval approve \
  --instance-id <processInstanceId> \
  --task-id <taskId> \
  --remark "工单核实，同意部署" \
  --format json

# Step 4b: 拒绝审批
dws oa approval reject \
  --instance-id <processInstanceId> \
  --task-id <taskId> \
  --remark "环境信息不完整，请补充后再提交" \
  --format json

适用命令oa approval list-pending / detail / tasks / approve / reject

场景6：团队考勤核查 — Attendance 考勤

🕘 需求：核查团队今日出勤情况，外勤人员是否正常打卡

# 查询个人考勤记录
dws attendance record get \
  --user <USER_ID> \
  --date 2026-04-30 \
  --format json

# 批量查询团队排班
dws attendance shift list \
  --users userId1,userId2,userId3 \
  --start 2026-04-27 \
  --end 2026-05-03 \
  --format json

# 查看考勤统计摘要
dws attendance summary \
  --user <USER_ID> \
  --date "2026-04-30 09:00:00" \
  --format json

# 查看考勤规则（打卡范围、弹性时间）
dws attendance rules --date 2026-04-30 --format json

适用命令attendance record get / shift list / summary / rules

场景7：运维数据报表 — Aitable AI表格

📊 需求：查询或管理存储监控数据的 AI 表格

# 搜索"服务器监控"相关的 AI 表格
dws aitable base search --query "服务器监控" --format json

# 获取表格详情
dws aitable base get --base-id <BASE_ID> --format json

# 列出数据表中的记录
dws aitable record list \
  --base-id <BASE_ID> \
  --table-id <TABLE_ID> \
  --format json

# 查看仪表盘（图表可视化）
dws aitable dashboard get \
  --base-id <BASE_ID> \
  --dashboard-id <DASHBOARD_ID> \
  --format json

适用命令aitable base search / record list / dashboard get

场景8：日志文件归档 — Drive 云盘

💾 需求：将服务器日志文件上传到钉钉云盘，供团队下载

# Step 1: 获取上传信息
dws drive upload-info \
  --file-name "server-error.log" \
  --file-size <字节数> \
  --format json
# → 返回 uploadId 和上传 URL

# Step 2: 上传文件到 OSS
curl -X PUT -T "server-error.log" "<上传URL>"

# Step 3: 提交上传
dws drive commit \
  --file-name "server-error.log" \
  --file-size <字节数> \
  --upload-id <uploadId> \
  --format json

# 下载文件
dws drive download --file-id <dentryUuid> --format json

适用命令drive upload-info / commit / download

场景9：故障复盘文档管理 — Doc 钉钉文档

📝 需求：创建或更新故障复盘文档，写入 Markdown 内容

# 搜索故障复盘相关文档
dws doc search --query "故障复盘 4月" --format json

# 读取文档内容
dws doc node get --node-id <NODE_ID> --format json

# 创建新文档
dws doc node create \
  --title "【故障复盘】2026-04-30 数据库故障" \
  --folder-id <FOLDER_ID> \
  --format json

# 写入文档块
dws doc block create \
  --node-id <NODE_ID> \
  --block-type "text" \
  --content "## 故障时间\n2026-04-30 13:00 - 14:30" \
  --format json

适用命令doc search / node get / node create / block create

场景10：团队日报汇总 — Report 日志

📝 需求：按模板收集团队日报，统计提交情况

# 查看可用的日志模板
dws report template list --format json

# 按模板创建日报
dws report create \
  --template-id <TEMPLATE_ID> \
  --content "今日完成：修复CPU告警阈值配置\n明日计划：优化数据库索引" \
  --format json

# 查看收件箱（收到的日报）
dws report inbox --format json

# 查看已发送的日志
dws report list-sent --format json

# 查看日志详情
dws report detail --report-id <REPORT_ID> --format json

适用命令report create / inbox / list-sent / detail

场景11：告警邮件转发确认 — Mail 邮箱

📧 需求：搜索告警邮件，确认工单通知是否已发送

# 搜索关键词邮件
dws mail search \
  --keyword "CPU Alert Production" \
  --start "2026-04-30T00:00:00+08:00" \
  --end "2026-04-30T23:59:59+08:00" \
  --format json

# 查看邮件详情
dws mail get --mail-id <MAIL_ID> --format json

# 发送邮件
dws mail send \
  --to "ops-team@company.com" \
  --subject "【告警确认】服务器 CPU 异常已处理" \
  --content "告警时间：13:00\n处理时间：13:45\n处理人：张三\n结论：已修复。" \
  --format json

适用命令mail search / get / send

场景12：值班交接 AI 听记 — Minutes AI听记

🎙️ 需求：上传会议录音，自动生成转写和摘要，提取待办

# 获取听记列表
dws minutes list --format json

# 获取听记摘要和待办
dws minutes get \
  --minutes-id <MINUTES_ID> \
  --format json

# 获取思维导图
dws minutes mindmap --minutes-id <MINUTES_ID> --format json

# 获取发言人列表
dws minutes speaker-list --minutes-id <MINUTES_ID> --format json

适用命令minutes list / get / mindmap / speaker-list

场景13：快速找人 — Contact 通讯录

👤 需求：知道手机号或姓名，快速找到 userId 用于发消息或派任务

# 通过手机号查 userId
dws contact user get-by-mobile --mobile 13800138000 --format json

# 搜索用户（按姓名）
dws contact user search --keyword "张煜" --format json

# 获取当前用户信息
dws contact user get-self --format json

# 查询部门成员
dws contact department members --dept-id 1 --format json

# 搜索共同群（知道几个人的花名，查共同在哪些群）
dws chat search-common --nicks "张煜,李四" --format json

适用命令contact user get-by-mobile / user search / chat search-common

场景14：群消息审计 — Chat 消息管理

🔍 需求：审计某群在某时间段内的所有消息，排查异常发言

# 搜索群
dws chat search --query "运维故障处理" --format json

# 查看群成员
dws chat group members list --id <openconversation_id> --format json

# 按时间范围拉取群消息
dws chat message list \
  --group <openconversation_id> \
  --time "2026-04-30 00:00:00" \
  --format json

# 关键词搜索消息
dws chat message search \
  --keyword "异常" \
  --group <openconversation_id> \
  --start "2026-04-30T00:00:00+08:00" \
  --end "2026-04-30T23:59:59+08:00" \
  --format json

# 拉取 @我的消息
dws chat message list-mentions \
  --start "2026-04-30T00:00:00+08:00" \
  --end "2026-04-30T23:59:59+08:00" \
  --format json

# 导出群聊消息（脚本）
python scripts/chat_export_messages.py \
  --query "运维故障处理" \
  --time "2026-04-30 00:00:00"

适用命令chat message list / search / list-mentions / scripts/chat_export_messages.py

五、危险操作清单与安全规范

以下操作不可逆或高影响，AI 执行前必须先展示操作摘要并获得用户明确同意：

产品 命令 风险说明
aitable base delete / table delete / field delete / record delete 删除后数据无法恢复
calendar event delete / room delete / participant delete 参与者同步收到取消通知
chat group members remove / message recall-by-bot 成员被踢出无法自动恢复，消息撤回不可逆
doc block delete 文档块永久删除
ding message recall DING 消息撤回
oa approval revoke / approval reject 撤销自己发起的审批；拒绝他人审批
todo task delete 待办永久删除
minutes replace-text 全文批量替换转写与摘要
操作前确认流程：
Step 1 → 展示操作摘要（操作类型 + 目标对象 + 影响范围）
Step 2 → 用户明确回复确认（如 "确认" / "好的"）
Step 3 → 加 --yes 执行命令

八、总结

DWS CLI 将钉钉 14 大产品能力统一封装为一致的命令行接口，特别适合以下运维场景：

  • 告警通知：Webhook 机器人 + DING 消息，确保 P0 故障 100% 触达
  • 任务管理：Todo 待办派发 + OA 审批加速工单流转
  • 团队协作：Calendar 值班排班 + Report 日报收集
  • 数据管理：Aitable AI表格 监控报表 + Drive 云盘文件归档
  • 审计合规：Chat 消息审计 + Attendance 考勤核查
  • 知识沉淀：Doc 钉钉文档 故障复盘 + Minutes AI听记 会议纪要

结合 OpenClaw 的 Cron 定时任务能力，可以将日常运维操作自动化，彻底告别重复性手工劳动。
