🤔 为什么不再适合用 GitHub Pages？

GitHub Pages 很好，但它是为纯静态内容设计的。随着我们的需求增长，它的局限性越来越明显：

⚠️ GitHub Pages 的局限
  • ❌ 无法运行后端代码 — 没法做任务看板的动态交互、API 调用
  • ❌ 无法做服务端渲染 — SEO 和性能都受限
  • ❌ 无法实现认证授权 — 没法做登录、个人数据保护
  • ❌ 无法长连接 — WebSocket、实时通知都做不了
  • ❌ 提交-等待-刷新 — 每次改动都要 git push，网络不好时很痛苦
✅ 迁移到自有服务器的优势
  • 完整的 Linux 环境，想装什么装什么
  • 本地编辑，即时预览，效率更高
  • 支持任何后端技术（Node.js / Python / Go / Rust）
  • 可以跑定时任务、守护进程
  • 真正的域名 + HTTPS + 企业邮箱
  • 可以跑数据库（Redis / PostgreSQL / MySQL）

⚙️ 技术选型

考虑到服务器配置（4核8G / 40G存储 / 上海）和实际需求：

组件选择理由
Web 服务器 Nginx 高性能反向代理，静态文件处理能力强，省资源
博客框架 Hexo Markdown 写作，静态生成，本地编辑，无需数据库
任务看板 Todoist API / 自建 轻量级，可以用文件存储 JSON 即可
动态功能 Node.js + Express 轻量、和前端配合好、Elaine 就是 Node.js 环境
数据库 SQLite / 文件JSON 数据量小，零配置，无需单独服务进程
容器化 Docker 环境隔离，一键部署，版本回滚
HTTPS Let's Encrypt (Certbot) 免费、自动续期
💡 为什么选 Hexo 而不是自建博客系统？

Hexo 是静态站点生成器，每次写完 Markdown 直接本地预览，确认没问题后部署到服务器。无需数据库、无需 PHP/Python 运行时，纯静态 HTML + Nginx，性能和安全性都很好。

Nginx 路由规则

server {
    listen 443 ssl;
    server_name your-domain.com;

    # 博客静态文件
    location /blog/ {
        alias /var/www/hexo/public/;
        try_files $uri $uri/ =404;
    }

    # 任务看板
    location /kanban/ {
        alias /var/www/kanban/;
        try_files $uri $uri/ =404;
    }

    # API 服务 (Node.js)
    location /api/ {
        proxy_pass http://localhost:3000/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
    }

    # OpenClaw Dashboard
    location /openclaw/ {
        proxy_pass http://localhost:18789/;
        proxy_set_header Host $host;
    }
}

🌐 域名与 HTTPS

1️⃣获取域名（可选，暂时用 IP）

  • 阿里云万网 / 腾讯云 / Cloudflare 都可以
  • 一年几十块钱.xyz .online 域名最便宜
  • 如果只是自己用，可以先不用域名

2️⃣配置 HTTPS（Let's Encrypt 免费）

# 安装 Certbot
sudo apt update
sudo apt install certbot python3-certbot-nginx

# 获取证书（假设域名是 example.com）
sudo certbot --nginx -d example.com

# 自动续期测试
sudo certbot renew --dry-run
💡 如果没有域名怎么办？

可以先直接用 http://你的服务器IP 访问，只是浏览器会显示不安全。等有域名后再配置 HTTPS。

🚀 部署流程

日常部署工作流（博客为例）

# 1. 本地编辑（服务器上直接操作）
ssh user@your-server
cd /var/www/blog

# 2. 写文章
hexo new "我的新文章"
vim source/_posts/我的新文章.md

# 3. 预览（本地 4000 端口）
hexo server

# 4. 确认没问题后，生成静态文件
hexo generate

# 5. Nginx 自动提供静态文件，无需重启

使用 Git 做版本管理（可选）

# 在本地初始化 Git 仓库
git init
git add .
git commit -m "update blog"

# 推送到服务器（在服务器上配置 Git hooks 实现自动部署）
# 或者：用 rsync 同步文件

Docker 部署（推荐）

# docker-compose.yml
version: '3.8'
services:
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./data/www:/usr/share/nginx/html
      - ./data/nginx.conf:/etc/nginx/nginx.conf
      - ./data/certbot/conf:/etc/letsencrypt
      - ./data/certbot/www:/var/www/certbot
    depends_on:
      - nodejs

  nodejs:
    image: node:18-alpine
    working_dir: /app
    volumes:
      - ./data/app:/app
    command: node server.js
    ports:
      - "3000:3000"

服务器直连 vs 本地开发

方式优点缺点
服务器直连 直接预览，Elaine 能直接操作文件 需要保持 SSH 连接
