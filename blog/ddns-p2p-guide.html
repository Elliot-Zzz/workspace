🔍 背景：为什么要用 DDNS 做点对点访问

很多人家里有服务器，电信宽带没有公网IP，每次想在公司或者外面访问家里的服务器，都要：

  • 联系运营商开放入方向
  • 或者走内网穿透中继，速度很慢

其实有个更简单的方法——DDNS 动态域名 + 路由器端口映射，可以实现真正的点对点直连，速度快、配置简单。

💡 DDNS 点对点访问的前提

你的宽带需要获取到真实的公网IP。如何判断：

  • 登录路由器，看 WAN 口 IP
  • 访问 IP138 对比
  • 一致 → 可以用 DDNS；不一致（大内网）→ 需要其他方案

🔧 DDNS 工作原理

什么是 DDNS？

DDNS（Dynamic Domain Name System） = 动态域名系统。

普通 DNS 是"静态的"：域名 → 固定IP，IP 变了就失效。

DDNS 是"动态的"：域名 → 随时变化的IP，IP 变了自动更新。

DDNS 点对点访问流程

┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│  家里路由器   │      │  DDNS 服务商  │      │   你的电脑   │
│  (服务器在内网)│      │   (云端)      │      │   (在外面)   │
└──────┬──────┘      └──────┬──────┘      └──────┬──────┘
       │                    │                    │
       │  1. 检测到IP变化    │                    │
       │───────────────────>│                    │
       │                    │                    │
       │  2. 更新域名解析     │                    │
       │                    │                    │
       │<────────────────────────────────────────│
       │    3. 域名 → 你的公网IP                  │
       │────────────────────────────────────────>│
       │                    │                    │
       │  4. 路由器端口映射    │                    │
       │<────────────────────────────────────────│
       │                    │                    │
       │  5. 直连服务器！     │                    │
       │<────────────────────────────────────────│

关键点：全程直连，无中继，速度和在家里访问一样快。

📝 完整实施步骤

Step 1：确认路由器支持 DDNS

登录路由器管理界面，查找：

  • 高级设置 → 外部管理 → DDNS
  • 路由设置 → 动态DNS

主流路由器都支持：小米、华为、华硕、TP-Link 等。

Step 2：注册 DDNS 服务商

有两种选择：

  • 免费DDNS：花生壳、No-IP、Dynu（域名较长，如 xxx.oicp.net）
  • 自有域名：腾讯云/阿里云解析（需要自己域名，推荐！）

本文重点介绍自有域名 + 腾讯云解析的方法。

Step 3：在路由器中配置 DDNS

以腾讯云解析为例，路由器 DDNS 配置：

配置项 示例值 说明
服务商 腾讯云 / AliDDNS 根据路由器支持的选项选择
域名 home.yourdomain.com 你自己的域名
API密钥 SecretId/SecretKey 腾讯云控制台获取
更新间隔 5 分钟 检测IP变化的频率

Step 4：配置端口映射（端口转发）

在路由器中找到"端口映射"或"虚拟服务器"：

配置项 示例值 说明
外部端口 22022 外网访问用的端口（建议改非标准端口）
内部IP 192.168.1.100 服务器的局域网IP
内部端口 22 服务器上SSH的实际端口
协议 TCP 通常选TCP即可

Step 5：测试访问

# 在外网电脑或手机上测试
ssh -p 22022 user@home.yourdomain.com

🌐 自有域名 + 腾讯云解析配置（重点）

为什么推荐用自己的域名？

  • ✅ 域名短好记（如 home.example.com 而不是 xxx.oicp.net）
  • ✅ 可以绑SSL证书做HTTPS
  • ✅ 域名是自己的，不受DDNS服务商限制

Step 1：获取腾讯云 API 密钥

  1. 登录 腾讯云 DNSPod 控制台
  2. 进入"密钥管理"
  3. 创建密钥，获得 ID 和 Token
⚠️ 重要：保存好密钥！

Token 只显示一次，丢失需要重新创建。

Step 2：添加域名解析记录

  1. 在 DNSPod 控制台添加你的域名（如 example.com）
  2. 添加子域名记录：
主机记录 记录类型 记录值 备注
home A 当前公网IP 家庭服务器主地址
@ A 当前公网IP 根域名（可选）

主机记录填 home，你的访问地址就是 home.example.com

Step 3：路由器 DDNS 配置

登录路由器，找到 DDNS 设置，填写：

配置项 填写内容
服务商 DNSPod（腾讯云）或 AliDNS（阿里云）
域名 home.example.com
SecretId Step 1 获取的 ID
SecretKey Step 1 获取的 Token

Step 4：验证配置

# 检查域名是否解析正确
nslookup home.example.com

# 应该返回你的公网IP

⚖️ DDNS vs VPN 优缺点对比

对比项 DDNS + 端口映射 VPN（WireGuard/OpenVPN）
速度 🚀 极快（直连，无中继） 🚀 快（直连 P2P）
配置难度 ⭐ 简单（路由器点点点） ⭐⭐⭐ 中等（需要安装配置客户端）
依赖 只需要浏览器 需要安装 VPN 客户端软件
多设备管理 每个端口单独映射 加入VPN后访问所有服务
适用场景 SSH、远程桌面、特定服务 需要访问内网所有资源的场景
安全性 ⚠️ 需自行配置防火墙 ✅ VPN 本身加密
成本 免费（用自己的域名） 免费（软件本身）

⚠️ DDNS 的缺点详解（重点！）

1. 公网IP变化问题

  • 问题：国内家宽IP通常是动态的，断电重拨会变
  • 影响：DDNS 更新有延迟（通常5分钟），变化期间无法访问
  • 缓解：设置较短更新间隔；使用多个域名备份

2. 端口被封风险 ⚠️（最大缺点）

  • 问题：国内防火墙会扫描并封禁常见服务端口
  • 影响：SSH(22)、RDP(3389)、HTTP(80)、HTTPS(443) 容易被封
  • 表现：从外部无法连接，但路由器WAN口IP没变
  • 缓解：
    • 改用高位端口（如 22022、43344）
    • 使用 VPN 客户端的固定端口
    • 联系运营商申请解封（通常被封难解）
🚨 重要提醒

如果常用端口（22、3389、80、443）被封，DDNS 方案基本失效。建议一开始就使用高位端口，并准备 VPN 作为备份方案。

3. 无公网IP完全无法使用

  • 问题：NAT、大内网、4G/5G 移动网络没有公网IP
  • 影响：DDNS 无法指向任何地址
  • 缓解：使用内网穿透工具（frp、花生壳中继模式）或 Tailscale/ZeroTier

4. 安全性暴露

  • 问题：端口暴露在公网，会受到扫描和攻击
  • 影响：SSH 暴力破解、漏洞利用
  • 缓解：
    • 使用密钥登录，禁止密码
    • 安装 fail2ban 防止暴力破解
    • 限制来源IP（防火墙）

5. 多层路由访问困难

  • 问题：如果宽带有多层 NAT（如 移动铁通）
  • 影响：无法做端口映射
  • 判断：登录路由器查看 WAN IP，和 IP138 对比不一致就是多层 NAT

6. 域名备案问题

  • 问题：国内服务器域名需要备案
  • 影响：如果域名指向国内服务器，80/443 端口可能无法访问
  • 缓解：使用高位端口；域名指向海外服务器
💡 总结：DDNS 的适用场景

DDNS 适合有稳定公网IP、需求简单（只需要SSH/远程桌面）、不想装客户端的场景。

如果你是：

  • 需要访问多个服务 → VPN 更方便
  • 没有公网IP → 内网穿透或 Tailscale
  • 担心安全 → VPN 加密隧道更安全
  • 需要HTTPS → DDNS + Nginx 反向代理

🔐 安全配置建议

1. 修改SSH默认端口

# 编辑 SSH 配置
sudo nano /etc/ssh/sshd_config

# 找到 Port 22，改为
Port 22022

# 重启SSH
sudo systemctl restart sshd

2. 使用密钥登录，禁止密码

# 生成本地密钥
ssh-keygen -t ed25519

# 复制公钥到服务器
ssh-copy-id -p 22022 user@home.yourdomain.com

# 编辑SSH配置禁用密码
sudo nano /etc/ssh/sshd_config
# 添加或修改：
PasswordAuthentication no
PubkeyAuthentication yes

3. 使用 fail2ban 防止暴力破解

# 安装
sudo apt install fail2ban -y

# 启动
sudo systemctl enable fail2ban
sudo systemctl start fail2ban

4. 防火墙限制来源IP（推荐）

# 只允许特定IP访问SSH
sudo ufw allow from 你的固定IP to any port 22022

# 或只允许内网
sudo ufw allow from 192.168.1.0/24 to any port 22022

📝 总结

DDNS 方案评价

维度 评分 说明
易用性 ⭐⭐⭐⭐⭐ 路由器点点就完事
速度 ⭐⭐⭐⭐⭐ 点对点直连，无中继
成本 ⭐⭐⭐⭐⭐ 免费，用自己域名
安全性 ⭐⭐ 端口暴露，需额外防护
可靠性 ⭐⭐⭐ IP变化、被封风险
适用性 ⭐⭐⭐ 需要有公网IP
💡 推荐方案
  • 首选：DDNS + 高位端口 + 密钥登录（简单快速）
  • 备选：WireGuard VPN（安全但需要客户端）
  • 最后：Tailscale/内网穿透（无公网IP时）
