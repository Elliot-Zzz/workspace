🔍 背景：为什么需要安全的远程访问？

当我们部署 OpenClaw Gateway 在服务器上后，自然希望能从任何地方访问它的 Dashboard。但是，如果直接开放访问，就会面临安全风险。

本文目标：在不购买域名的情况下，实现安全的 HTTPS 远程访问。

🌐 HTTP vs HTTPS：为什么 HTTP 不安全？

HTTP 的工作方式

HTTP（HyperText Transfer Protocol）是"超文本传输协议"，用于浏览器和服务器之间的通信。

HTTP 的致命问题：数据在网络中以明文传输，任何中间人都能窃听！

HTTP 传输过程示意

浏览器 ──[明文数据]──> 服务器

这意味着：

  • ❌ 用户名/密码 可被窃取
  • ❌ Token/会话 可被劫持
  • ❌ 业务数据 可被窥探

HTTPS 的工作方式

HTTPS = HTTP + TLS（Transport Layer Security），在传输层对数据进行加密。

浏览器 ──[加密数据]──> 服务器
  │                         │
  └─ TLS 握手 ─────────────┘
  (验证证书、协商密钥、加密通信)
特性 HTTP HTTPS
数据传输 明文 ❌ 加密 ✅
身份验证 无 ❌ 证书验证 ✅
安全性 不安全 安全
端口 80 443

🚇 SSH 隧道：临时的安全方案

在我们配置 HTTPS 之前，为什么需要用 SSH 映射到本地 (127.0.0.1:18789)？

原因分析

  1. HTTP 明文传输：直接通过公网 IP 访问 HTTP，Token 会被明文发送，极其危险
  2. 无身份验证：HTTP 不验证服务器身份，可能被中间人攻击
  3. SSH 隧道提供加密：SSH 本身是加密的，所以本地映射相对安全
SSH 隧道的工作原理：
本地端口 ──[SSH 加密隧道]──> 服务器

浏览器访问本地 127.0.0.1:18789，数据通过 SSH 加密隧道传输到服务器。

SSH 隧道的缺点

  • ❌ 需要 SSH 客户端，不是所有设备都有
  • ❌ 配置相对繁琐
  • ❌ 无法从手机等设备直接访问

🔒 HTTPS 工作原理：TLS/SSL 加密

TLS 握手过程

  1. Client Hello：客户端告诉服务器支持的加密套件
  2. Server Hello：服务器选择加密方式，返回证书
  3. 证书验证：客户端验证服务器证书（自签名需手动信任）
  4. 密钥协商：双方生成会话密钥
  5. 加密通信：后续数据用会话密钥加密传输

证书类型

类型 价格 需要域名 浏览器信任
Let's Encrypt (免费) 免费 ✅ 是 自动信任
自签名证书 免费 ✅ 否 需手动信任
商业证书 付费 是 自动信任

⚙️ Nginx + 自签名证书：实现 HTTPS 访问

由于我们没有域名，使用自签名证书是最佳方案。虽然浏览器不自动信任，但手动信任一次后就可以正常使用。

步骤一：安装 Nginx

# Ubuntu/Debian
sudo apt update
sudo apt install nginx

# 验证安装
nginx -v

步骤二：生成自签名 SSL 证书

# 创建证书存放目录
sudo mkdir -p /etc/nginx/ssl

# 生成私钥和证书
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
    -keyout /etc/nginx/ssl/openclaw.key \
    -out /etc/nginx/ssl/openclaw.crt

# 参数说明：
# -x509: 自签名证书格式
# -days 365: 有效期一年
# -newkey rsa:2048: 2048位 RSA 密钥

步骤三：配置 Nginx 反向代理

# 创建配置文件
sudo nano /etc/nginx/sites-available/openclaw
            
server {
    listen 443 ssl;
    server_name _;

    # SSL 证书配置
    ssl_certificate /etc/nginx/ssl/openclaw.crt;
    ssl_certificate_key /etc/nginx/ssl/openclaw.key;

    # 安全配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    location / {
        # 反向代理到 OpenClaw Gateway
        proxy_pass http://127.0.0.1:18789;
        proxy_http_version 1.1;
        
        # WebSocket 支持
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        
        # 传递真实 IP
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name _;
    return 301 https://$server_name$request_uri;
}

步骤四：启用配置

# 创建符号链接
sudo ln -s /etc/nginx/sites-available/openclaw /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx

步骤五：访问并信任证书

重要：首次访问 https://你的IP:443 时，浏览器会显示"您的连接不是私密连接"。

点击"高级" → "继续前往（不安全）"即可。只需要信任一次，之后就正常了。

✅ 安全配置清单

1. 强密码/Token

  • Token 长度至少 32 位
  • 包含大小写字母、数字、特殊字符
  • 定期更换

2. 防火墙配置

# 只开放必要端口
sudo ufw allow 443/tcp   # HTTPS
sudo ufw allow 22/tcp    # SSH
sudo ufw enable

3. 阿里云安全组

在阿里云控制台 → 安全组 → 配置规则：

端口 协议 授权对象
443 TCP 0.0.0.0/0 或你的IP
22 TCP 你的IP

4. 定期更新

  • Nginx 安全更新
  • OpenClaw 版本更新
  • 系统补丁

📝 总结

配置完成后：
✅ 浏览器访问 https://你的公网IP:443
✅ 数据全程加密传输
✅ 无需 SSH 隧道
✅ 支持手机等所有设备

访问方式对比

方式 安全性 便捷性 适用场景
HTTP + 公网IP ❌ 极不安全 最方便 仅本地测试
HTTP + SSH隧道 ✅ 安全 较繁琐 临时访问
HTTPS + 自签名 ✅ 安全 方便 生产环境推荐
HTTPS + Let's Encrypt ✅✅ 安全 最方便 有域名时首选
